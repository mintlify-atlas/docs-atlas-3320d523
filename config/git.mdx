The Git configuration file (`.gitconfig`) defines your Git identity, default behaviors, and convenient command shortcuts.

## Overview

The configuration includes:
- User identity settings
- Default branch name
- Global gitignore file
- Helpful command aliases

## Configuration File

```ini .gitconfig
[user]
	email = maclong9@icloud.com
	name = Mac
[init]
	defaultBranch = main
[core]
  excludesFile = "$HOME/.gitignore"
[alias]
	co = checkout
	br = branch
	st = status
	lg = log --oneline --graph --decorate
```

## Sections

### User Identity

```ini
[user]
	email = maclong9@icloud.com
	name = Mac
```

Defines your Git identity used in commit metadata. This information appears in commit history and is essential for attribution.

<Note>
You'll want to update these values with your own name and email address during setup.
</Note>

### Default Branch

```ini
[init]
	defaultBranch = main
```

Sets the default branch name to `main` for new repositories. This aligns with GitHub's current default and industry best practices.

### Global Exclude File

```ini
[core]
  excludesFile = "$HOME/.gitignore"
```

References a global `.gitignore` file that applies to all Git repositories on your system. This is useful for ignoring OS-specific files and editor artifacts without polluting individual project `.gitignore` files.

The `.gitignore` file includes:

```gitignore
*.DS_Store
*.{swp,swo}
gh
github-copilot/
node_modules/
zed/prompts/
zed/themes/
```

**What's Ignored:**
- `*.DS_Store` - macOS directory metadata files
- `*.{swp,swo}` - Vim temporary and swap files
- `gh` - GitHub CLI binary (if locally built)
- `github-copilot/` - GitHub Copilot cache directory
- `node_modules/` - Node.js dependencies
- `zed/prompts/` - Zed editor custom prompts
- `zed/themes/` - Zed editor custom themes

### Aliases

```ini
[alias]
	co = checkout
	br = branch
	st = status
	lg = log --oneline --graph --decorate
```

Defines shortcuts for commonly used Git commands:

## Usage Examples

### Enhanced Log Visualization

The `lg` alias is particularly useful for visualizing branch history:

```bash
$ git lg
* a1b2c3d (HEAD -> main) Add new feature
* d4e5f6g Update documentation
*   h7i8j9k Merge branch 'feature-x'
|\
| * k9l0m1n Implement feature X
|/
* n2o3p4q Initial commit
```

The `--oneline` flag shows abbreviated commits, `--graph` displays branch structure, and `--decorate` shows branch and tag names.

## Installation

The Git configuration is symlinked during setup:

```bash
ln -sf "$DOTFILES_DIR/.gitconfig" "$HOME/.gitconfig"
```

<Warning>
Make sure to update the `[user]` section with your personal information before committing code.
</Warning>
